HandshakeFinished

Fields

§id: ConnectionId

Identifier of the connection.

§expected_peer_id: Option<PeerId>

Parameter that was passed to ChainNetwork::add_single_stream_connection or ChainNetwork::add_multi_stream_connection.

§peer_id: PeerId

Actual PeerId of the connection.

A connection that was added with ChainNetwork::add_single_stream_connection or ChainNetwork::add_multi_stream_connection has now finished its handshake phase. Its PeerId is now known which certainty.

GossipConnected

Peer we are now connected to.

§chain_id: ChainId

Chain of the gossip connection.

§kind: GossipKind

Which kind of gossip link is concerned.

§role: Role

Role the node reports playing on the network.

§best_number: u64

Height of the best block according to this node.

§best_hash: [u8; 32]

Hash of the best block according to this node.

Now connected to the given peer for gossiping purposes.

This event can only happen as a result of a call to ChainNetwork::gossip_open.

Error that happened.

Error in the protocol in a connection, such as failure to decode a message. This event doesn’t have any consequence on the health of the connection, and is purely for diagnostic purposes.

BlocksRequestIn

Fields

§peer_id: PeerId

Remote that has sent the request.

§chain_id: ChainId

Index of the chain concerned by the request.

§config: BlocksRequestConfig

Information about the request.

§substream_id: SubstreamId

Identifier of the request. Necessary to send back the answer.

A remote has sent a request for blocks.

Can only happen for chains where ChainConfig::allow_inbound_block_requests is true.

You are strongly encouraged to call ChainNetwork::respond_blocks.

§

RequestInCancel

Fields

§substream_id: SubstreamId

Identifier of the request.

This SubstreamId is considered dead and no longer valid.

A remote is no longer interested in the response to a request.

Calling ChainNetwork::respond_identify, ChainNetwork::respond_blocks, or similar will now panic.
